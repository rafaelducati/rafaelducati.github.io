Case study 1 - Uselets Minhas Medidas

Mar 21, 2016

Overview

O outlet Uselets é um e-commerce que trabalha com marcas nacionais e internacionas do mercado fashion. Como UX e webdesigner da equipe de desenvolvimento do Uselets, desde sua fase de startup, minha responsabilidade era desenvolver soluções para melhorar a encontrabilidade do produto pelo usuário.

Problema

Após 12 meses de operação, o departamento de trocas e devoluções registrou um excesso de suas atividades. Por se tratar de um contexto multimarcas, havia muita diferenciação entre os tamanhos dos produtos e as medidas não estavam devidamente esclarecidas nas páginas de produto. A insatisfação do usuário foi detectada pela crescente incidência das transações de trocas ou devoluções, que implicava também em mais trabalho para este departamento.

Os usuários precisavam de uma maneira mais fácil de encontrar produtos que se encaixavam com seu tamanho, e além disso, se sentirem seguros para comprar estes produtos com suas medidas exatas.

Processo

O processo começou com a identificação da necessidade de exibir mais informações para os tamanhos dos produtos, além dos clássicos P, M e G. Isso significava, em parte, estender os dados de tamanho dos produtos cruzando com os novos dados do usuário.

Em conjunto com o departamento de produto e compras, foram definidas novas medidas gerais para cada produto:

altura, busto/tórax cintura quadril comprimento calçado

Como estes dados eram inexistentes até então na base de dados, fiquei com a responsabilidade de idealizar as interfaces do fluxo de entrada destes novos dados a serem inputados pelo usuário.

A ideia era esclarecer, com mais precisão, as medidas ideais para cada usuário. Após reuniões com o departamentos de TI, de produto e de compras, comecei a desenvolver o Minhas Medidas.

Após passar para o papel as idéias, criei um pequeno wireframe low level utilizando o Axure.

Após aprovado o wireframe, montei um pequeno protótipo navegável com InvisionApp e parti para a etapa de validação com usuários. Validei o protótipo com 5 usuários diferentes, fazendo uma pesquisa de guerrilha. Esse método incluída heavy users de ecommerce de moda. Observar suas reações foi fundamental para a validação. A princípio foram descobertos alguns pontos interessantes:

- A maioria dos usuários não tinha conhecimento de suas medidas específicas.

- Algumas das medidas não estavam esclarecidas o suficiente ao usuário.

- Não havia associação visual para cada medida e o usuário aumentando o tempo de inserção dos dados.

Alterações:

- Inserção de passos curtos através de tópicos, orientando o usuário ao preenchimento.

- Inserção de um infográfico associado diretamenta aos inputs de entrada de cada medida.

- Modificação por gênero, dependente do modelo do infográfico.

- Inserção de um video curto esclarecendo cada medida e como fazer a medição.

UI design final

Após os dados obtidos com o protótipo navegável, parti para o conceito visual das telas utilizando o Sketch.

Após o preenchimento das medidas, uma vitrine com todos os produtos concatenados com estas informações é exibida em seguida. Um algoritmo foi deselvolvido para aumentar as exibições destes produtos pelas vitrines do site de acordo com os dados do usuário.

O passo seguinte foi a criação de um selo indicativo de produto com medidas exatas do usuário, a ser exibido na vitrine e na página de produto. Produtos com este selo possuem as medidas corretas do usuário e e já com a medida selecionada para ser imediatamente comprado pelo mesmo.

Uma tela de edição de medidas foi derivada da tela inicial de input para que o usuário corrija ou atualize suas informações.

Testes com usuários

A maioria dos meus testes com usuários estavam com pessoas ao redor do escritório. Aproveitei oportunidades para abordá-las por 5 minutos para fazer os testes e então receber sugestões sobre o protótipo, assim que elas concluissem o cadastro das medidas.

Resultados

- Usuários sentem-se seguros e convencidos a comprar produtos que sabem possuir suas medidas corretas.

- As métricas mostraram uma melhor taxa de visitantes novos.

- Os números mostraram uma melhor taxa de conversão após a implementação.

- Houve também um aumento de produtos favoritados por usuário.

- As trocas e devoluções bem como as reclamações sobre tamanhos de produtos entregues diminuíram consideravelmente.
